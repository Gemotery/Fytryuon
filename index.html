Особенности Fytryuon

  • Все Особенности: 1. Префикс Uoion_ для встроенных конструкций: • Большинство встроенных команд, функций, блоков управления и ключевых слов языка Fytryuon начинаются с префикса Uoion_. Это создает характерный стиль языка, делает его легко узнаваемым и отделяет его от других языков. Это также помогает структурировать внутренние механизмы языка и облегчает их идентификацию. Эта особенность придает языку уникальность. 2. Многоступенчатое присваивание переменных: • Присваивание значения переменным в Fytryuon происходит не напрямую, а в несколько этапов, используя последовательность команд Uoion_meaning (имя переменной), Holdet_meaning, и meaning = (значение). Этот необычный процесс присваивания придает языку своеобразие и выделяет его среди языков с более традиционной структурой присваивания. Это также может служить своеобразным "ритуалом" для подчеркивания значимости присваивания в языке. 3. Блочная структура с использованием двоеточий: • Fytryuon использует двоеточие (:) для обозначения начала блоков кода, таких как условные операторы (if:, else:), блоки рандома (Uoion_random: start:, Uoion_random: stop:), математические блоки (:math_start :, :math_stop :), и т.д. Этот подход к структурированию кода отличает Fytryuon от языков, которые используют скобки или другие символы для обозначения блоков. Двоеточие в данном случае играет роль своеобразного "маркера начала" блока кода. 4. Встроенная поддержка библиотек: • Fytryuon имеет встроенную поддержку библиотек, таких как math для математических вычислений и window для создания оконных приложений. Эти библиотеки устанавливаются и импортируются с помощью специальных команд, что позволяет расширять возможности языка, не усложняя его базовый синтаксис. Эта особенность показывает, что Fytryuon нацелен на предоставление готовых решений для различных задач, не ограничиваясь базовыми возможностями языка. В заключение: Эти четыре особенности делают Fytryuon уникальным и запоминающимся языком программирования. Они определяют его стиль и отличают его от других языков. Мы выбрали эти особенности, чтобы придать языку индивидуальность и отразить его особенную философию. Как тебе эти особенности? Ты считаешь, что они точно отражают сущность Fytryuon? Может быть, ты хочешь добавить что-то еще?

Синтаксис Fytryuon

Это Весь Синтаксис языка программирования Fytryuon<code> для примеров кода. 


                // Пример кода на Fytryuon
                •  Использование Uoion_ префикса.
•  Многоступенчатое присваивание переменных.
•  Двоеточия для обозначения начала блоков.
•  / для комментариев.
•  [ ] для типов данных и математических операций
•  ( ) для параметров функций и вывода.
1. Uoion_cycle_operator: start :: начало блока цикла.
2. Inf_cycle: инициализация бесконечного цикла.
3. :start.inf.cycle : начало самого цикла, именно отсюда начнется повторение.
4. Sumbyt_print: "Hello from infinite cycle!": Команда вывода текста.
5. later (uoion) [1]: задержка в 1 секунду.
/"Импорт математической библиотеки"/
Import_math

/"Начало блока математики"/
:math_start :

    /"Объявление переменной x и присваивание ей значения 10"/
    Uoion = variable, Variable = x
    Uoion_meaning x, Holdet_meaning, meaning = [ 10 ]

    /"Объявление переменной y и присваивание ей значения 5"/
    Uoion = variable, Variable = y
    Uoion_meaning y, Holdet_meaning, meaning = [ 5 ]

    /"Вычисляем сумму x + y и выводим результат"/
    Uoion = math
        Sumbyt_print: "x + y = "
        Sumbyt_print: [ x + y ]

:math_stop :
6. Uoion_cycle_operator: stop :: конец блока цикла. •  Библиотека для создания окон (window):
  •  Установка: Unstall_window
  •  Начало работы с окном: Start_window
  •  Установка названия окна: Uoion_name_window = (название)
  •  Установка размера окна: Uoion_size_window = (размер)
  •  Установка цвета заливки: Uoion_filling_window
  •  Установка цвета заливки: filling_window (R, G, B)
•  Цикл для бесконечного повторения (Inf_cycle):
  •  Начало блока цикла: Uoion_cycle_operator: start :
  •  Инициализация цикла: Inf_cycle
  •  Начало бесконечного цикла: :start.inf.cycle
  •  Конец блока цикла: Uoion_cycle_operator: stop :
